# LiquidationsPalletDeals.com — Next.js 15 Site

A full Next.js 15 + TypeScript + Tailwind CSS implementation of the wholesale101.com site structure: a liquidation-pallet marketplace with category browsing, deal tiers, manifest-aware pallet detail pages, and standard info pages.

## Stack

- **Next.js 15** (App Router, React Server Components)
- **React 19**
- **TypeScript 5.6** (strict)
- **Tailwind CSS 3.4**
- **next/image** for AVIF/WebP delivery and lazy loading

## Quick start

```bash
npm install
npm run dev        # http://localhost:3000
npm run typecheck  # tsc --noEmit
npm run lint
npm run build      # production build
npm start          # serve the build
```

## Editing content

- **Pallets** — `src/data/pallets.ts`. Real lots are stored as compact raw records (SKU, title, price, retail value, units, cost/unit, condition, manifest) and `decorate()` derives the slug, discount %, blurb, detail list, tags, and `/images/products/<SKU>-*.webp` paths. Each lot generates a `/pallets/[slug]` page via `generateStaticParams`; category and deal-tier listings derive from the same data. The catalog currently holds 30 real lots (page 1 of the source collection's 177) — add more by appending raw records and dropping the matching `<SKU>-1/2` images into `public/images/products/`.
- **Categories** — `src/data/categories.ts`. Drives the header mega-menu, footer, home grid, and per-category pages.
- **FAQ** — `src/data/faq.ts`. Grouped by `Buying`, `Manifests`, `Shipping`, `Returns`, `Account`.
- **Site identity** — `src/lib/site.ts` (name, email, phone, address, socials).

## Assets & image pipeline

All imagery, logos, the favicon, the social/OG image, and the homepage video are the **real Wholesale101 assets**, migrated from the source site and optimized for the web.

- **Originals** are staged in `_recon/assets/` (git-ignored — large source files kept locally for re-optimization).
- **`scripts/optimize-assets.mjs`** (uses `sharp`) reads the staged originals plus the extracted catalog JSON and emits right-sized, EXIF-corrected WebP into `public/images/` (named by SKU for products, by purpose for brand/marketing/category art), plus `public/favicon.png`, `public/og-image.png`, and `public/videos/hero.mp4`.

```bash
node scripts/optimize-assets.mjs   # re-run after adding originals to _recon/assets
```

To add product imagery: append the raw lot to `src/data/pallets.ts` (its image paths are derived from the SKU), drop the source photos into `_recon/assets/`, extend the product map in the optimize script if needed, and re-run it. `next/image` then serves responsive AVIF/WebP with explicit `sizes` at request time.

The legal/shipping page copy lives as structured JSON in `src/content/` (`terms.json`, `privacy.json`, `shipping.json`) and renders through `src/components/LegalContent.tsx`.

## Performance and accessibility built in

- Static generation for every category, deal tier, and pallet detail page via `generateStaticParams` — every product page is pre-rendered at build time
- `next/image` with explicit `sizes` for AVIF/WebP and responsive `srcset`
- Skip-to-content link, `aria-*` on accordions and nav, focus rings on every interactive control
- Mobile-first responsive layout with a slide-down mobile nav
- No client JS on legal, FAQ-list-only, or category pages — only the accordion/form/header use `"use client"`
- Tailwind purged via `content: ["./src/**/*.{ts,tsx}"]`
- Custom 404 page (`not-found.tsx`)

## Deployment

The project is Vercel-ready as-is:

```bash
vercel        # link
vercel --prod # deploy
```

For other hosts, `npm run build` produces a `.next/` directory you can run with `npm start` behind any Node host (Render, Fly.io, Railway, AWS App Runner, etc.). For static hosts, swap `next.config.ts` to `output: "export"` — note that this disables the Next image optimizer.
